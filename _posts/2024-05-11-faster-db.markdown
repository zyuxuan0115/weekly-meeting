### How to merge functions in a container

#### Merging 2 real serverless functions

![d3](/assets/2024-05-16/call-graph-3.png)

- the functions being chosen: write-home-timeline & social-graph-get-followers

#### The step to build a merged function

![d1](/assets/2024-05-16/d1.png)

#### Issue 1

- The building process takes a very long time: about 20 minutes
	+ and this is just for merging 2 functions
	+ merging all `.ll` files 
- The building process needs at least 30GB
	+ can merge functions on another machine
	+ since the final thing we need is just a docker image on docker hub

![s1](/assets/2024-05-16/s1.png)

all these IRs contains more than 500 thousands of lines

#### Issue 2
We have 2 clusters and each cluster has a redis DB
- Before merge
	+ <strong>Caller</strong>: on cluster 1, load/store things on DB 1
		* redis internal access: `@sn-redis-master.openfaas-fn.svc.cluster.local:6379`
	+ <strong>Callee</strong>: on cluster 2, load/store things on DB 2
		* redis internal access: `@sn-redis-master.openfaas-fn.svc.cluster.local:6379`
- After merge
	+ Caller + Callee: on cluster 1
		* <strong>Caller</strong>: still internal access 
		* <strong>Callee</strong>: external redis access
			- `@130.127.133.16:6379`
